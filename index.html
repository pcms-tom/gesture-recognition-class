01 · Computer Vision

AI 如何看見手

MediaPipe Hands 是一個已訓練好的手部追蹤模型，它可以分析畫面並輸出一組可計算的關鍵點座標。

從影像到程式資料

這一課要建立的核心觀念

電腦一開始只看到很多像素，並不知道哪裡是手腕、哪裡是食指。MediaPipe Hands 的工作，是把「一張圖片」轉成「一份手部骨架資料」：0 到 20 號共 21 個關節點，每個點都有對應的 x、y、z 座標。

AI 模型負責看圖推理，程式負責判斷規則。例如「食指舉起」不是模型直接得出，而是程式用食指尖和食指根的 y 座標比較出來。

這裡「推理」的意思是：模型收到目前這一幀畫面後，計算每個關節最可能出現的位置。它每秒重複很多次，因此畫面中的骨架會跟著你的手移動。

  • 攝影機畫面會連續送入模型，因此左邊的座標會一直更新。
  • 模型輸出的是「landmarks list」，也就是一串關鍵點資料。
STEP 01

攝影機擷取畫面

每一幀都是一張圖片，通常每秒大約 30 張。

STEP 02

模型找出手掌

AI 先判斷畫面中哪裡可能有手，避免把整張照片都當成手來分析。

STEP 03

標記 21 個關節

每個點都有 x、y、z，代表手部骨架。

STEP 04

程式寫判斷規則

比較座標之間的關係，就能判斷食指、剪刀手或握拳。

02 · Landmarks

21 個關鍵點

MediaPipe Hands 把一隻手拆成 21 個可追蹤的點，例如手腕 0、五根手指尖 4 / 8 / 12 / 16 / 20，以及指根 5 / 9 / 13 / 17

0
手腕 Wrist很多判斷會用它做參考點，例如大拇指向左或向右。
4
大拇指尖 Thumb Tip大拇指常用 X 座標判斷，因為它多半是水平伸出。
8
食指尖 Index Tip最常拿來控制遊戲角色、游標或按鈕。
12
中指尖 Middle Tip和食指一起判斷「剪刀手」。
5
食指根 Index MCP食指尖比食指根高，通常代表食指舉起。

關鍵點不是「圖片標籤」，而是可以計算的骨架

觀察「手部地圖」。每個黃色或綠色圓點都代表一個 landmark（關鍵點），也就是一個關節位置；藍色線條代表關節之間的連接。當你把手移動、旋轉、伸直或彎曲時，這些點的位置會跟著改變。

程式最常做的事情，是拿兩個點比較。例如食指尖 8 和食指根 5，如果 8 比 5 更高，就表示食指很可能伸直。這比直接分析整張照片簡單很多，也更適合初學者理解。

  • 先找「指尖」：4、8、12、16、20，這些點最能代表手勢外形。
  • 再找「指根」：5、9、13、17，用來判斷手指是否真的伸直。
  • 最後找「參考點」：0 手腕，可以判斷手掌方向或大拇指方向。

03 · Coordinates

座標系統

每個 landmark 都有 x、y、z 座標，x 表示左右，y 表示上下，z 表示相對深度。注意座標不是像素，是 0 到 1 的比例。

X 變大 →
Y 變大 ↓
[8] x=0.63
y=0.24
# 第一隻手的 21 個點
hand = results.multi_hand_landmarks[0]
lm = hand.landmark

# 食指尖的位置
index_tip = lm[8]
x = index_tip.x # 0 = 左，1 = 右
y = index_tip.y # 0 = 上，1 = 下

# 如果要換成 640 x 480 畫面的像素
px = x * 640
py = y * 480

為什麼不用像素直接表示？

如果你的電腦畫面是 640 × 480，食指尖可能在 x = 320px；但同一個位置換到 1280 × 960 的畫面，就會變成 x = 640px。像素會因畫面大小而改變，不適合直接拿來寫通用規則。

MediaPipe 用 0.0 到 1.0 的比例 表示位置：x = 0.5 永遠是畫面中央，y = 0.25 永遠是畫面上方四分之一。這樣你的手勢規則就不會因為電腦螢幕或攝影機解析度改變而失效。

  • x 越小越靠左，x 越大越靠右。
  • y 越小越靠上，y 越大越靠下，這點和一般數學座標不同。
  • 要實際畫圖時，才把比例乘上實際寬高換成像素。

04 · Python Gesture Logic

手勢識別

手勢識別就是把關節關係寫成條件，AI 給出座標，程式負責定義條件，當條件成立就判定某個手勢出現。

☝️ 食指舉起
index_up = lm[8].y < lm[5].y
✌️ 剪刀手
peace = (lm[8].y < lm[5].y and
         lm[12].y < lm[9].y)
✊ 握拳
fist = (lm[8].y > lm[5].y and
        lm[12].y > lm[9].y and
        lm[16].y > lm[13].y)
🤏 捏合
pinch = abs(lm[4].x - lm[8].x) < 0.05

從「單一條件」到「完整手勢」

條件判斷就是讓電腦問一個可以回答 True 或 False 的問題，例如「食指尖是否比食指根更高？」如果 True 就代表食指很可能舉起。

一個可靠的手勢通常不只看一根手指。例如「剪刀手」如果只寫食指和中指舉起，張開手掌也會符合；所以還要補上無名指、小指放下，才會更精準。

這就是手勢識別最像設計的地方：你不是只寫程式，而是在設計一個可被機器理解的規則。好的規則要夠清楚、容易做出來，也要盡量避免和其他手勢混淆。

  • and 代表 AND：所有條件都成立，結果才成立。
  • or 代表 OR：其中一個條件成立，結果就成立。
  • 先寫容易成功的版本，再慢慢加條件讓它更精準。

05 · Practice

互動實作

互動實驗室包含攝影機、即時數據、答題、程式編輯器與手勢闖關，嘗試按以下順序完成

實作路線圖

實作不是照抄，是把觀察變成規則

進入互動實驗室後，先不要急著寫程式。第一步是做幾個手勢，觀察左側即時資料怎麼變；第二步才是讀題目，確認自己真的理解 x、y、指尖、指根的意思。

手勢識別條件：

  • 先用一個條件測試，例如只判斷食指是否舉起。
  • 如果太容易誤判，再增加第二或第三個條件。
  • 完成後請同學試做，看看你的規則是否穩定。
任務 A · 觀察座標

移動食指，記錄 X / Y 座標如何變化。

任務 B · 解讀條件

回答程式碼理解題，弄清楚 AND、OR、比較大小如何影響手勢判斷。

任務 C · 自創手勢

設計一個同學能重現、但又不太容易誤判的手勢。

# Python：把你觀察到的規則寫成函式
def detect_my_gesture(lm):
  index_up = lm[8].y < lm[5].y
  middle_down = lm[12].y > lm[9].y
  return index_up and middle_down
