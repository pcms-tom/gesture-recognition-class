認識 21 個關鍵點

MediaPipe 怎麼「看見」你的手

傳統程式要辨認「手」需要寫好幾百行規則。AI 的做法不同：
Google 用幾十萬張手部圖片訓練了一個模型，它學會了把每隻手分析成 21 個關鍵點（Landmarks），每個點代表一個手部關節。

每個關鍵點都有 X、Y、Z 座標：X 是左右位置，Y 是上下位置，Z 是深度（離鏡頭遠近）。
座標用 0.0 ~ 1.0 表示，0 = 畫面最左/最上，1 = 畫面最右/最下（注意攝影機鏡頭會左右翻轉）。
# 只需要三行，就能拿到所有 21 個關鍵點
lm = results.multi_hand_landmarks[0].landmark
index_tip = lm[8] # 食指尖
print(index_tip.x, index_tip.y) # 例如：0.52, 0.34

讀取座標數據

把 0.0～1.0 轉換成真實像素位置

MediaPipe 輸出的每個關鍵點都有三個屬性：.x（左右）、.y（上下）、.z（深度）。
數值範圍是 0.0 ~ 1.0 的比例值：0 = 最左/最上，1 = 最右/最下。

透過索引編號取得想要的點，例如 lm[8] 就是食指尖，lm[0] 是手腕。
# multi_hand_landmarks[0] = 第一隻識別到的手
lm = results.multi_hand_landmarks[0].landmark

# lm[8] = 食指尖，取其屬性
print(lm[8].x) # 水平位置 0~1
print(lm[8].y) # 垂直位置 0~1（0=最上）
print(lm[0].y) # 手腕垂直位置

識別手勢

用座標之間的大小關係判斷手的動作

AI 輸出的是數字，不是「食指舉起了」這種文字。
但我們可以自己寫條件：如果食指尖（Y 值）比食指根（Y 值）更小，就代表食指向上伸直。

記住：Y 越小 = 越高。所以「手指往上伸 = 指尖 Y < 根部 Y」。
# 食指舉起的判斷邏輯
tip = lm[8] # 食指尖
base = lm[5] # 食指根
is_up = tip.y < base.y # 尖比根高 = 舉起
